## 一、阶段三目标

阶段三的核心目标是完成 **Agent Runtime 与 Claude Adapter**，让用户可以通过图形界面启动 Agent、发送消息、接收流式输出。交付一个可用的 Agent 对话界面，为后续 Event Bus 与 Timeline 提供数据源。

### 1.1 核心功能

| 功能 | 说明 |
|------|------|
| Claude CLI 集成 | 通过子进程调用 Claude CLI，获取流式输出 |
| 会话管理 | 支持多轮对话，通过 session_id 续接会话 |
| 流式输出解析 | 解析 stream-json 格式，实时显示 Agent 输出 |
| Agent 状态管理 | 管理 Agent 的生命周期状态 |
| 对话界面 | 用户可以发送消息，查看 Agent 回复 |
| 错误处理 | 处理 CLI 不可用、超时、进程异常等情况 |

### 1.2 交付物

- 可启动的 Claude Agent 对话界面
- 支持多轮对话，保持上下文
- 实时流式显示 Agent 输出
- 支持停止 Agent 执行

1. **每轮对话一个进程**：使用 `claude -p` 模式，每条消息启动一个进程
2. **session 续接**：通过 `--resume session_id` 保持多轮对话上下文

4. **进程生命周期管理**：优雅停止（SIGTERM）+ 强制停止（SIGKILL）
5. **事件驱动架构**：通过 IPC 事件推送 Agent 输出到渲染进程

# 续接会话

Claude CLI 的 `stream-json` 格式每行一个 JSON 对象，常见类型：

```typescript

// 工具结果
{ type: 'tool_result', tool_use_id: '...', content: '...' }

// 错误
{ type: 'error', error: '...' }

// 最终结果
{ type: 'result', result: '...', session_id: '...' }
```

## 五、关键逻辑说明

### 5.1 Claude CLI 集成策略

**问题**：如何集成 Claude CLI，支持多轮对话？

**解决方案**：采用 "每轮对话一个进程 + session 续接" 模式。

```typescript
// electron/main/agent/claude-adapter.ts
async send(message: string): Promise<void> {
  if (this.status === 'running') {
    throw new Error('Agent is already running');
  }

  this.setStatus('starting');

  const args = ['-p', '--output-format', 'stream-json'];

  // 权限模式
  if (this.options.permissionMode) {
    args.push('--permission-mode', this.options.permissionMode);
  }

  // 续接会话
  if (this.sessionId) {
    args.push('--resume', this.sessionId);
  }

  // 消息内容
  args.push(message);

  this.currentProcess = spawn(getClaudeCommand(), args, {
    cwd: this.options.workspacePath,
    stdio: ['pipe', 'pipe', 'pipe'],
    env: this.buildEnv(),
  });

  this.setStatus('running');

  // ... 处理输出
}
```

**优点**：
- 进程生命周期简单，一条消息一个进程
- `--resume` 保持会话上下文
- 更容易处理超时、取消、重试

**踩坑点**：
- Windows 上命令是 `claude.cmd`，不是 `claude`
- 需要设置 `shell: true` 支持跨平台
- 需要传递环境变量（API_BASE_URL、API_KEY）

### 5.2 流式输出解析

**问题**：如何解析 Claude CLI 的 stream-json 输出？

**解决方案**：逐行解析 JSON，处理缓冲区。

```typescript
// electron/main/agent/claude-adapter.ts
let outputBuffer = '';

this.currentProcess.stdout?.on('data', (data: Buffer) => {
  outputBuffer += data.toString();
  const lines = outputBuffer.split('\n');
  outputBuffer = lines.pop() || '';

  for (const line of lines) {
    if (!line.trim()) continue;
    try {
      const json = JSON.parse(line);
      this.handleStreamEvent(json);
    } catch {
      this.emitOutput({ type: 'text', data: line, timestamp: Date.now() });
    }
  }
});
```

**事件处理**：

```typescript
private handleStreamEvent(json: Record<string, unknown>): void {
  // 提取 session_id
  if (json.session_id && typeof json.session_id === 'string') {
    this.sessionId = json.session_id;
  }

  const type = json.type as string;

  switch (type) {
    case 'content_block_delta':
      // 文本内容流
      this.emitOutput({ type: 'text', data: json, timestamp: Date.now() });
      break;
    case 'tool_use':
      // 工具调用
      this.emitOutput({ type: 'tool_use', data: json, timestamp: Date.now() });
      break;
    case 'tool_result':
      this.emitOutput({ type: 'tool_result', data: json, timestamp: Date.now() });
      break;
    case 'error':
      this.emitOutput({ type: 'error', data: json, timestamp: Date.now() });
      break;
    case 'result':
      // 最终结果
      this.emitOutput({ type: 'result', data: json, timestamp: Date.now() });
      break;
  }
}
```

**踩坑点**：
- 输出可能被分割成多行，需要缓冲处理
- JSON 解析失败时，作为普通文本处理
- 需要处理空行和无效数据

### 5.3 进程生命周期管理

**问题**：如何优雅停止 Agent 进程？

**解决方案**：SIGTERM 优雅停止 + SIGKILL 强制停止。

```typescript
// electron/main/agent/claude-adapter.ts
async stop(): Promise<void> {
  if (this.currentProcess) {
    // 发送 SIGTERM，请求优雅停止
    this.currentProcess.kill('SIGTERM');

    // 给 3 秒优雅退出，否则强制 kill
    this.killTimer = setTimeout(() => {
      if (this.currentProcess) {
        this.currentProcess.kill('SIGKILL');
        this.currentProcess = null;
      }
    }, AGENT_SIGKILL_DELAY_MS);

    this.setStatus('idle');
  }
}

destroy(): void {
  if (this.killTimer) {
    clearTimeout(this.killTimer);
  }
  this.stop();
  this.onOutputCallback = null;
  this.onStatusCallback = null;
  this.onErrorCallback = null;
}
```

**踩坑点**：
- 必须先发送 SIGTERM，给进程清理的机会
- 设置超时，防止进程挂起
- 应用退出时必须清理所有子进程

### 5.4 会话续接机制

**问题**：如何支持多轮对话，保持上下文？

**解决方案**：通过 session_id 续接会话。

```typescript
// electron/main/agent/claude-adapter.ts
private sessionId: string | null = null;

// 在 handleStreamEvent 中提取 session_id
private handleStreamEvent(json: Record<string, unknown>): void {
  if (json.session_id && typeof json.session_id === 'string') {
    this.sessionId = json.session_id;
  }
  // ... 处理其他事件
}

// 在 send 方法中使用 session_id
async send(message: string): Promise<void> {
  const args = ['-p', '--output-format', 'stream-json'];

  if (this.sessionId) {
    args.push('--resume', this.sessionId);
  }

  args.push(message);

  // ... 启动进程
}
```

**优点**：
- 自动保持多轮对话上下文
- 不需要手动管理历史消息
- Claude CLI 负责上下文裁剪

**踩坑点**：
- session_id 可能过期，需要处理续接失败
- 需要提供 resetSession 方法，允许重新开始

### 5.5 IPC 事件推送

**问题**：如何将 Agent 输出实时推送到渲染进程？

**解决方案**：通过 IPC 事件推送。

```typescript
// electron/main/ipc/agent.ts
ipcMain.handle(IPC_CHANNELS.AGENT_START, async (event, options) => {
  const win = BrowserWindow.fromWebContents(event.sender);

  adapter.onOutput((outputEvent) => {
    win?.webContents.send(IPC_CHANNELS.AGENT_OUTPUT, outputEvent);
  });

  adapter.onStatusChange((status) => {
    win?.webContents.send(IPC_CHANNELS.AGENT_STATUS, { status });
  });

  return { success: true, version: availability.version };
});
```

**渲染进程监听**：

```typescript
// src/stores/agent.ts
const unsubOutput = window.api.on(IPC_CHANNELS.AGENT_OUTPUT, (event: unknown) => {
  get()._handleOutputEvent(event as AgentOutputEvent);
});

const unsubStatus = window.api.on(IPC_CHANNELS.AGENT_STATUS, (data: unknown) => {
  const { status } = data as { status: string };
  get()._updateStatus(status as AgentState['status']);
});
```

**踩坑点**：
- 必须在 preload 中允许 AGENT_OUTPUT 和 AGENT_STATUS 事件
- 需要在组件卸载时取消监听
- 需要处理窗口关闭时的清理

### 5.6 错误处理策略

**问题**：如何处理各种错误情况？

**解决方案**：分层错误处理。

```typescript
// 1. CLI 不可用
const availability = await adapter.checkAvailability();
if (!availability.available) {
  return { success: false, error: availability.error };
}

// 2. 进程启动失败
this.currentProcess.on('error', (err) => {
  this.setStatus('error');
  this.onErrorCallback?.(err);
});

// 3. 进程超时
const timeout = setTimeout(() => {
  this.stop();
  reject(new Error('Agent execution timed out'));
}, this.options.timeoutMs);

// 4. 进程异常退出
this.currentProcess.on('close', (code) => {
  if (code !== 0) {
    this.setStatus('error');
    reject(new Error(`Claude process exited with code ${code}`));
  }
});
```

**错误类型**：
- **CLI 不可用**：Claude CLI 未安装或版本不兼容
- **进程启动失败**：权限问题、路径问题
- **进程超时**：执行时间过长
- **进程异常退出**：内存不足、信号中断
- **JSON 解析失败**：输出格式异常

### 5.7 配置集成

**问题**：如何从配置读取 Claude Profile？

**解决方案**：在 AGENT_START 时读取配置，创建 adapter。

```typescript
// electron/main/ipc/agent.ts
ipcMain.handle(IPC_CHANNELS.AGENT_START, async (event, options) => {
  // 读取配置
  const fullConfig = configStore.store;
  const decrypted = decryptApiKeys(fullConfig);
  const profiles = decrypted.claude.profiles;
  const activeId = decrypted.claude.activeProfileId;
  const activeProfile = profiles.find(p => p.id === activeId);

  adapter = new ClaudeAdapter({
    workspacePath: options.workspacePath,
    model: activeProfile?.model || options.model,
    permissionMode: options.permissionMode ?? 'default',
    apiBaseUrl: activeProfile?.baseUrl,
    apiKey: activeProfile?.apiKey,
  });

  // ... 检测 CLI 可用性，绑定事件
});
```

**踩坑点**：
- API Key 需要解密后再使用
- 需要处理配置不存在的情况
- 需要支持覆盖配置（如手动指定 model）

## 七、常见问题

### Q1: Claude CLI 不可用？

**可能原因**：
- Claude CLI 未安装
- API Key 配置错误
- 网络连接问题

**解决方案**：
1. 终端运行 `claude --version` 检查安装
2. 在设置页面检查 Claude 配置
3. 检查 `ANTHROPIC_BASE_URL` 和 `ANTHROPIC_API_KEY` 环境变量

### Q2: Agent 无法启动？

**可能原因**：
- Claude CLI 不可用
- 工作区路径无效
- 权限问题

**解决方案**：
1. 检查 Claude CLI 可用性
2. 确认工作区路径存在
3. 检查进程权限

### Q3: 流式输出不显示？

**可能原因**：
- JSON 解析失败
- 事件监听未绑定
- 渲染进程未更新

**解决方案**：
1. 检查 Claude CLI 输出格式
2. 确认 AGENT_OUTPUT 事件监听已绑定
3. 检查 store 是否正确更新

### Q4: 多轮对话丢失上下文？

**可能原因**：
- session_id 未正确保存
- 续接失败
- Claude CLI 不支持续接

**解决方案**：
1. 检查 session_id 是否正确提取
2. 确认 `--resume` 参数正确传递
3. 检查 Claude CLI 版本是否支持续接

### Q5: Agent 超时？

**可能原因**：
- 执行时间过长
- 网络延迟
- Claude CLI 卡住

**解决方案**：
1. 调整 `AGENT_DEFAULT_TIMEOUT_MS` 常量
2. 检查网络连接
3. 手动停止 Agent，重新开始
