## 一、阶段二目标

阶段二的核心目标是实现 **Workspace 与文件系统**，让用户可以打开文件夹、浏览文件树、编辑文件、保存文件。交付一个类似轻量 VS Code 的工作区界面，为后续 Agent Runtime 提供文件操作基础。

### 1.1 核心功能

| 功能 | 说明 |
|------|------|
| 打开文件夹 | 通过系统对话框选择本地文件夹 |
| 文件树展示 | 递归显示目录结构，支持展开/折叠 |
| 文件读取 | 读取文件内容并在编辑器中显示 |
| 文件写入 | 保存编辑器中的修改 |
| 文件监听 | 监听文件系统变化，实时更新文件树 |
| Monaco Editor | 代码编辑器，支持语法高亮 |
| 多标签管理 | 同时打开多个文件，标签页切换 |
| 最近工作区 | 记录最近打开的工作区，快速访问 |

### 1.2 交付物

- 类似轻量 VS Code 的工作区界面
- 支持文件浏览、打开、编辑、保存
- 支持多文件标签页
- 支持最近工作区记录

## 五、关键逻辑说明

### 5.1 文件树递归构建

**问题**：如何高效构建文件树，避免扫描 `node_modules` 等大目录？

**解决方案**：

```typescript
// electron/main/ipc/fs.ts
const IGNORED_DIRS = new Set([
  'node_modules', '.git', 'dist', '.next', '__pycache__', '.DS_Store',
  'dist-electron', '.nocodeflow'
]);

async function buildFileTree(
  dirPath: string,
  maxDepth: number = FILE_TREE_MAX_DEPTH,
  currentDepth: number = 0,
): Promise<FileNode[]> {
  if (currentDepth >= maxDepth) return [];

  const entries = await fs.readdir(dirPath);
  const nodes: FileNode[] = [];

  for (const entry of entries) {
    if (IGNORED_DIRS.has(entry)) continue;

    const fullPath = path.join(dirPath, entry);
    const stat = await fs.stat(fullPath);

    const node: FileNode = {
      name: entry,
      path: fullPath,
      type: stat.isDirectory() ? 'directory' : 'file',
      size: stat.size,
      modifiedAt: stat.mtimeMs,
    };

    if (stat.isDirectory()) {
      node.children = await buildFileTree(fullPath, maxDepth, currentDepth + 1);
    }

    nodes.push(node);
  }

  // 目录优先，然后按名称排序
  nodes.sort((a, b) => {
    if (a.type !== b.type) return a.type === 'directory' ? -1 : 1;
    return a.name.localeCompare(b.name);
  });

  return nodes;
}
```

**踩坑点**：
- 符号链接可能导致无限递归，需要检测
- 权限不足的目录需要跳过，不能抛出异常
- Windows 和 macOS 的路径分隔符不同，需要统一处理

### 5.2 原子写入

**问题**：写入过程中断会导致文件损坏吗？

**解决方案**：使用原子写入模式，先写临时文件，再重命名。

```typescript
// electron/main/ipc/fs.ts
ipcMain.handle(IPC_CHANNELS.FS_WRITE, async (_event, args) => {
  const { filePath, content } = args;
  
  // 写入临时文件
  const tmpPath = filePath + '.nocodeflow-tmp';
  await fs.writeFile(tmpPath, content, 'utf-8');
  
  // 原子重命名
  await fs.rename(tmpPath, filePath);
  
  return { success: true };
});
```

**优点**：
- 写入中断不会损坏原文件
- 重命名是原子操作，要么成功要么失败
- 临时文件在重命名前可以验证内容

### 5.3 路径安全校验

**问题**：如何防止符号链接攻击，确保文件操作在工作区内？

**解决方案**：使用 `fs.realpath` 解析真实路径，然后检查是否在工作区内。

```typescript
// electron/main/ipc/fs.ts
function isPathWithinWorkspace(filePath: string, workspaceRoot: string): boolean {
  const resolved = path.resolve(filePath);
  const resolvedRoot = path.resolve(workspaceRoot);
  return resolved === resolvedRoot || resolved.startsWith(resolvedRoot + path.sep);
}

// 读取文件时的安全检查
ipcMain.handle(IPC_CHANNELS.FS_READ, async (_event, args) => {
  const { filePath } = args;
  
  if (!currentWorkspaceRoot) {
    return { error: 'No workspace open' };
  }
  
  if (!isPathWithinWorkspace(filePath, currentWorkspaceRoot)) {
    return { error: 'Access denied: path is outside workspace' };
  }
  
  // 检查符号链接
  try {
    const realPath = await fs.realpath(filePath);
    if (!isPathWithinWorkspace(realPath, currentWorkspaceRoot)) {
      return { error: 'Access denied: symlink points outside workspace' };
    }
  } catch {
    // 文件不存在，检查父目录
    const parentDir = path.dirname(filePath);
    const realParent = await fs.realpath(parentDir);
    if (!isPathWithinWorkspace(realParent, currentWorkspaceRoot)) {
      return { error: 'Access denied: parent directory outside workspace' };
    }
  }
  
  // ... 读取文件
});
```

**踩坑点**：
- 必须同时检查原始路径和真实路径
- 文件不存在时需要检查父目录
- 符号链接可能指向工作区外的敏感文件

### 5.4 二进制文件检测

**问题**：如何避免在编辑器中打开二进制文件？

**解决方案**：读取文件头部，检查是否包含空字节。

```typescript
// electron/main/ipc/fs.ts
const BINARY_CHECK_BUFFER_SIZE = 8192;

async function isBinaryFile(filePath: string): Promise<boolean> {
  try {
    const fd = await fs.open(filePath, 'r');
    const buf = Buffer.alloc(BINARY_CHECK_BUFFER_SIZE);
    const { bytesRead } = await fd.read(buf, 0, BINARY_CHECK_BUFFER_SIZE, 0);
    await fd.close();

    for (let i = 0; i < bytesRead; i++) {
      if (buf[i] === 0) return true;
    }
    return false;
  } catch {
    return true; // 无法读取，假设为二进制
  }
}
```

**优点**：
- 快速检测，只读取前 8KB
- 准确率高，文本文件通常不包含空字节
- 错误处理完善，无法读取时默认为二进制

### 5.5 文件监听与防抖

**问题**：编辑器保存文件时会触发多次文件系统事件，如何避免频繁更新？

**解决方案**：使用 chokidar 的 `awaitWriteFinish` 选项进行防抖。

```typescript
// electron/main/ipc/fs.ts
import chokidar from 'chokidar';
import { WATCHER_THROTTLE_MS, WATCHER_STABILITY_THRESHOLD_MS } from '@shared/constants';

let watcher: chokidar.FSWatcher | null = null;

ipcMain.handle(IPC_CHANNELS.FS_WATCH, async (_event, args) => {
  const { dirPath } = args;
  
  if (watcher) {
    await watcher.close();
  }
  
  watcher = chokidar.watch(dirPath, {
    ignored: /(^|[\/\\])\../, // 忽略点文件
    persistent: true,
    ignoreInitial: true,
    awaitWriteFinish: {
      stabilityThreshold: WATCHER_STABILITY_THRESHOLD_MS, // 300ms
      pollInterval: WATCHER_THROTTLE_MS, // 100ms
    },
  });
  
  watcher.on('all', (event, path) => {
    const win = getMainWindow();
    if (win) {
      win.webContents.send(IPC_CHANNELS.FS_WATCH_EVENT, { type: event, path });
    }
  });
});
```

**踩坑点**：
- `awaitWriteFinish` 会延迟事件触发，但能避免重复事件
- 需要在应用退出时清理 watcher
- 忽略点文件（如 `.git`）避免不必要的事件

### 5.7 标签页状态管理

**问题**：如何管理多个打开的文件，跟踪未保存状态？

**解决方案**：使用 Zustand store 管理标签页状态。

```typescript
// src/stores/workspace.ts
interface TabInfo {
  path: string;
  name: string;
  content: string;
  originalContent: string; // 用于比较是否修改
  isDirty: boolean;
  language: string;
}

// 打开文件时
openFile: async (path: string) => {
  const { openTabs } = get();
  const existingTab = openTabs.find(t => t.path === path);
  
  if (existingTab) {
    // 已打开，切换到该标签
    set({ activeTabPath: path });
    return;
  }
  
  // 读取文件内容
  const result = await ipcInvoke<{ content: string }>(IPC_CHANNELS.FS_READ, { filePath: path });
  
  // 创建新标签
  const newTab: TabInfo = {
    path,
    name: path.split(/[\\/]/).pop() || path,
    content: result.content,
    originalContent: result.content,
    isDirty: false,
    language: getLanguageForFile(path),
  };
  
  set({
    openTabs: [...openTabs, newTab],
    activeTabPath: path,
  });
},

// 更新内容时
updateTabContent: (path: string, content: string) => {
  const { openTabs } = get();
  const updatedTabs = openTabs.map(tab => {
    if (tab.path === path) {
      return {
        ...tab,
        content,
        isDirty: content !== tab.originalContent,
      };
    }
    return tab;
  });
  set({ openTabs: updatedTabs });
},

// 保存文件时
saveFile: async (path: string) => {
  const { openTabs } = get();
  const tab = openTabs.find(t => t.path === path);
  if (!tab) return;
  
  await ipcInvoke(IPC_CHANNELS.FS_WRITE, { filePath: path, content: tab.content });
  
  // 更新标签状态
  const updatedTabs = openTabs.map(t => {
    if (t.path === path) {
      return { ...t, originalContent: t.content, isDirty: false };
    }
    return t;
  });
  set({ openTabs: updatedTabs });
},
```

**踩坑点**：
- 必须同时保存 `content` 和 `originalContent`，用于比较是否修改
- 保存后需要更新 `originalContent`，否则 `isDirty` 状态不正确
- 关闭未保存标签时需要弹确认对话框

### 5.8 最近工作区记录

**问题**：如何记录和显示最近打开的工作区？

**解决方案**：使用 electron-store 持久化最近工作区列表。

```typescript
// src/stores/workspace.ts
openWorkspace: async (dirPath: string) => {
  // 获取文件树
  const tree = await ipcInvoke<FileNode[]>(IPC_CHANNELS.FS_TREE, { rootPath: dirPath });
  
  // 更新最近工作区
  const config = await ipcInvoke<AppConfig>(IPC_CHANNELS.CONFIG_GET_ALL);
  const recentWorkspaces = config.recentWorkspaces || [];
  
  // 去重并插入到头部
  const updatedRecent = [
    dirPath,
    ...recentWorkspaces.filter(p => p !== dirPath),
  ].slice(0, MAX_RECENT_WORKSPACES);
  
  await ipcInvoke(IPC_CHANNELS.CONFIG_SET, 'recentWorkspaces', updatedRecent);
  
  // 更新状态
  set({
    rootPath: dirPath,
    workspaceName: dirPath.split(/[\\/]/).pop() || dirPath,
    isOpen: true,
    fileTree: tree,
    expandedPaths: new Set(),
    selectedPath: null,
  });
},
```

**踩坑点**：
- 需要去重，避免重复记录
- 限制最大数量，避免列表过长

## 七、常见问题

### Q1: 文件树不显示？

**可能原因**：
- 工作区路径无效
- 权限不足
- 忽略规则误匹配

**解决方案**：
1. 检查 `rootPath` 是否正确
2. 确认有目录读取权限
3. 检查 `IGNORED_DIRS` 是否包含目标目录

### Q2: 文件无法保存？

**可能原因**：
- 文件被其他程序占用
- 磁盘空间不足
- 权限问题

**解决方案**：
1. 检查文件是否被锁定
2. 确认磁盘空间
3. 检查文件写入权限

### Q3: 编辑器无法打开文件？

**可能原因**：
- 文件太大（超过 10MB）
- 二进制文件
- 文件编码问题

**解决方案**：
1. 检查文件大小限制
2. 确认文件是文本文件
3. 检查文件编码

### Q4: 文件监听不工作？

**可能原因**：
- chokidar 未正确安装
- 权限问题
- 防抖时间设置不当

**解决方案**：
1. 检查 `chokidar` 依赖
2. 确认有目录监听权限
3. 调整 `WATCHER_STABILITY_THRESHOLD_MS` 常量

### Q5: 主题切换不生效？

**可能原因**：
- CSS 变量未正确应用
