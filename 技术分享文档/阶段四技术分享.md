## 一、阶段四目标

阶段四的核心目标是实现 **Event Bus 与 Agent Timeline**，把 Agent 的底层输出转换成用户能理解的执行过程。用户能看到 Agent 正在：**分析项目 → 读取文件 → 修改文件 → 执行命令 → 生成结果**，而不是一堆原始 JSON 事件。

### 1.1 核心功能

| 功能 | 说明 |
|------|------|
| Event Bus | 事件总线，解耦事件生产与消费，支持多路监听 |
| Output Parser | 输出解析器，将原始事件解析为结构化 Timeline 条目 |
| TimelineBuilder | 时间线构建器，维护有序的 Timeline 条目列表 |
| Timeline UI | 时间线 UI 组件，展示工具调用、结果、状态 |
| Chat UI 集成 | 将 Timeline 条目集成到对话流中 |

### 1.2 交付物

- 用户能看到 Agent 调用了哪些工具（Read、Write、Bash 等）
- 用户能展开工具调用卡片查看输入参数和结果
- 流式文本正常显示
- 最终结果和错误清晰展示

### 3.2 设计模式

#### 发布-订阅模式

Event Bus 采用发布-订阅模式，解耦事件生产与消费：

```
事件生产者（Agent Store）
    ↓ emit()
Event Bus
    ↓ on()
事件消费者（TimelineBuilder、ChatUI）
```

#### Discriminated Union

TimelineEntry 使用 discriminated union 模式，通过 `kind` 字段区分类型：

```typescript
type TimelineEntry =
  | TextEntry        // kind: 'text'
  | ToolUseEntry     // kind: 'tool_use'
  | ToolResultEntry  // kind: 'tool_result'
  | ResultEntry      // kind: 'result'
  | SystemEntry      // kind: 'system'
  | ErrorEntry;      // kind: 'error'
```

### 4.2 核心模块职责

#### Event Bus (`src/lib/event-bus.ts`)

事件总线，负责：

- **订阅事件**：`on(eventType, listener)` 返回取消订阅函数
- **发布事件**：`emit(eventType, event)` 同步调用所有监听器
- **移除监听器**：`off(eventType)` 移除某类型的所有监听器
- **清除全部**：`clear()` 清除所有监听器

```typescript
interface EventBus {
  on<T = unknown>(eventType: string, listener: EventListener<T>): () => void;
  emit<T = unknown>(eventType: string, event: T): void;
  off(eventType: string): void;
  clear(): void;
  listenerCount(eventType: string): number;
}

// 单例实例
export const agentEventBus = createEventBus();
```

#### Output Parser (`src/lib/output-parser.ts`)

输出解析器，负责：

- **解析事件**：将 `AgentOutputEvent` 解析为 `TimelineEntry`
- **生成描述**：为工具调用生成人类可读的描述
- **类型映射**：将工具名称映射为中文显示名

```typescript
function parseOutputEvent(event: AgentOutputEvent): TimelineEntry | null {
  switch (event.type) {
    case 'text':
      return { kind: 'text', content: event.data.delta?.text, ... };
    case 'tool_use':
      return { kind: 'tool_use', toolName: event.data.name, ... };
    case 'tool_result':
      return { kind: 'tool_result', toolUseId: event.data.tool_use_id, ... };
    case 'result':
      return { kind: 'result', content: event.data.result, ... };
    case 'error':
      return { kind: 'error', message: event.data.message, ... };
    case 'system':
      return { kind: 'system', content: JSON.stringify(event.data), ... };
  }
}
```

#### TimelineBuilder (`src/lib/timeline-builder.ts`)

时间线构建器，负责：

- **流式文本合并**：将多个 text 事件合并为一个连续的文本
- **工具调用配对**：通过 tool_use_id 将 tool_use 和 tool_result 配对
- **状态管理**：维护条目的状态（pending → running → completed/error）

```typescript
class TimelineBuilder {
  private entries: TimelineEntry[] = [];
  private activeTextEntryId: string | null = null;
  private pendingToolUses = new Map<string, ToolUseEntry>();

  handleEvent(event: AgentOutputEvent): void {
    const entry = parseOutputEvent(event);
    if (!entry) return;

    switch (entry.kind) {
      case 'text':
        this.handleText(entry);
        break;
      case 'tool_use':
        this.handleToolUse(entry);
        break;
      case 'tool_result':
        this.handleToolResult(entry);
        break;
      case 'result':
        this.closeActiveText();
        this.entries.push(entry);
        break;
      case 'error':
        this.closeActiveText();
        this.entries.push(entry);
        break;
      case 'system':
        this.entries.push(entry);
        break;
    }
  }

  getEntries(): readonly TimelineEntry[] {
    return this.entries;
  }

  reset(): void {
    this.entries = [];
    this.activeTextEntryId = null;
    this.pendingToolUses.clear();
  }
}
```

## 五、关键逻辑说明

### 5.1 Event Bus 实现

**问题**：如何解耦事件生产与消费，支持多路监听？

**解决方案**：使用发布-订阅模式。

```typescript
// src/lib/event-bus.ts
export function createEventBus(): EventBus {
  const listeners = new Map<string, Set<EventListener>>();

  return {
    on<T = unknown>(eventType: string, listener: EventListener<T>): () => void {
      let set = listeners.get(eventType);
      if (!set) {
        set = new Set();
        listeners.set(eventType, set);
      }
      set.add(listener as EventListener);

      return () => {
        set!.delete(listener as EventListener);
        if (set!.size === 0) {
          listeners.delete(eventType);
        }
      };
    },

    emit<T = unknown>(eventType: string, event: T): void {
      const set = listeners.get(eventType);
      if (!set) return;
      // Snapshot to iterate — safe if listeners unregister during dispatch
      for (const listener of [...set]) {
        listener(event);
      }
    },

    off(eventType: string): void {
      listeners.delete(eventType);
    },

    clear(): void {
      listeners.clear();
    },

    listenerCount(eventType: string): number {
      return listeners.get(eventType)?.size ?? 0;
    },
  };
}
```

**优点**：
- 解耦事件生产与消费
- 支持多监听器
- 监听器注销安全（emit 期间注销不影响当次分发）

**踩坑点**：
- 必须在 emit 时复制监听器集合，避免迭代期间修改
- 取消订阅函数必须检查集合是否为空，自动清理
- 同步调用监听器，事件处理应快速返回

### 5.2 流式文本合并

**问题**：Claude 的流式输出会发送多个 text 事件，如何合并为连续的文本？

**解决方案**：维护 activeTextEntryId，追加到活跃条目。

```typescript
// src/lib/timeline-builder.ts
private handleText(entry: TextEntry): void {
  const active = this.activeTextEntry;
  if (active) {
    // 追加到活跃条目
    active.content += entry.content;
    active.timestamp = entry.timestamp;
  } else {
    // 创建新条目
    this.entries.push(entry);
    this.activeTextEntryId = entry.id;
  }
}

private closeActiveText(): void {
  if (!this.activeTextEntryId) return;
  const active = this.activeTextEntry;
  if (active) {
    active.isStreaming = false;
    active.status = 'completed';
  }
  this.activeTextEntryId = null;
}
```

**优点**：
- 避免创建大量小文本条目
- 保持文本的连续性
- 支持流式显示（isStreaming 标志）

**踩坑点**：
- 必须在 tool_use、result、error 事件前关闭活跃文本
- 需要处理空文本事件
- 时间戳需要更新为最新事件的时间戳

### 5.3 工具调用配对

**问题**：如何将 tool_use 和 tool_result 配对？

**解决方案**：通过 tool_use_id 关联。

```typescript
// src/lib/timeline-builder.ts
private pendingToolUses = new Map<string, ToolUseEntry>();

private handleToolUse(entry: ToolUseEntry): void {
  this.closeActiveText();
  this.pendingToolUses.set(entry.toolId, entry);
  this.entries.push(entry);
}

private handleToolResult(entry: ToolResultEntry): void {
  const pending = this.pendingToolUses.get(entry.toolUseId);
  if (pending) {
    // 配对成功，更新状态
    entry.toolName = pending.toolName;
    pending.status = entry.isError ? 'error' : 'completed';
    this.pendingToolUses.delete(entry.toolUseId);
  }
  this.entries.push(entry);
}
```

**优点**：
- 自动关联工具调用和结果
- 支持异步工具调用
- 处理配对失败的情况

**踩坑点**：
- 必须处理 tool_use_id 不存在的情况
- 需要从 tool_use 获取工具名称，补充到 tool_result
- 需要更新 tool_use 的状态（completed/error）

### 5.4 工具描述生成

**问题**：如何为工具调用生成人类可读的描述？

**解决方案**：根据工具名称和输入参数生成描述。

```typescript
// src/lib/output-parser.ts
const TOOL_DISPLAY_NAMES: Record<string, string> = {
  Read: '读取文件',
  Write: '写入文件',
  Edit: '编辑文件',
  Bash: '执行命令',
  Glob: '搜索文件',
  Grep: '搜索内容',
  WebFetch: '获取网页',
  WebSearch: '搜索网络',
  Agent: '调用子 Agent',
};

export function buildToolDescription(toolName: string, input: Record<string, unknown>): string {
  switch (toolName) {
    case 'Read':
      return `读取 ${input.file_path || '文件'}`;
    case 'Write':
      return `写入 ${input.file_path || '文件'}`;
    case 'Edit':
      return `编辑 ${input.file_path || '文件'}`;
    case 'Bash':
      return `执行 ${input.command || '命令'}`;
    case 'Glob':
      return `搜索 ${input.pattern || '文件'}`;
    case 'Grep':
      return `搜索 "${input.pattern || ''}"`;
    default:
      return TOOL_DISPLAY_NAMES[toolName] || toolName;
  }
}
```

**优点**：
- 快速了解 Agent 在做什么
- 支持中文显示
- 易于扩展新工具

**踩坑点**：
- 输入参数可能为空，需要默认值
- 工具名称可能未知，需要 fallback
- 描述可能过长，需要截断

## 七、常见问题

### Q1: Timeline 不显示？

**可能原因**：
- Event Bus 未正确初始化
- TimelineBuilder 未集成到 store
- TimelinePanel 组件未渲染

**解决方案**：
1. 检查 agentEventBus 是否正确创建
2. 确认 TimelineBuilder 在 initialize 中创建
3. 检查 ChatPanel 是否渲染 TimelinePanel

### Q2: 工具调用不显示？

**可能原因**：
- tool_use 事件未正确解析
- ToolUseEntryView 组件未渲染
- 消息类型不匹配

**解决方案**：
1. 检查 parseOutputEvent 是否正确解析 tool_use
2. 确认 MessageBubble 支持 tool_use 角色
3. 检查 toolEntry 是否正确填充

### Q3: 工具结果不显示？

**可能原因**：
- tool_result 事件未正确解析
- tool_use_id 不匹配
- ToolResultEntryView 组件未渲染

**解决方案**：
1. 检查 parseOutputEvent 是否正确解析 tool_result
2. 确认 tool_use_id 与 tool_use 事件匹配
3. 检查 TimelinePanel 是否正确查找 tool_result

### Q4: 流式文本不连续？

**可能原因**：
- text 事件未正确合并
- activeTextEntryId 未正确维护
- closeActiveText 调用时机不对

**解决方案**：
1. 检查 handleText 是否正确追加内容
2. 确认 activeTextEntryId 在创建新条目时设置
3. 检查 closeActiveText 是否在 tool_use、result、error 前调用

### Q5: 图标不显示？

**可能原因**：
- 工具名称不匹配
- lucide-react 未正确安装
- ToolIcon 组件未正确导入

**解决方案**：
1. 检查 TOOL_ICONS 映射是否包含工具名称
2. 确认 lucide-react 已安装
3. 检查 ToolIcon 组件是否正确导入
