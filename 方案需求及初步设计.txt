需求：
项目定位： 
为 Claude Code 套上一个图形界面，让非计算机用户也能方便使用 AI 编程能力。 
详细需求： 
    跨平台：支持 Windows 和 macOS。请评估是用一套代码跨平台（如 Electron、Tauri），还是需要分别开发两套原生应用？ 一键安装：用户下载后双击就能安装使用，像普通软件一样，不需要配置环境变量或安装 Node.js 等依赖。 对话界面：类似 Lobe Chat 的聊天风格，不是命令行黑框。界面上可以配置 API URL、API Key 等参数。 文件管理：像 VS Code 一样可以打开文件夹、查看文件树、浏览和编辑文件内容。 Agent 能力：底层内置 Claude Code，所有对话和代码操作都由 Claude Code Agent 完成。 扩展能力：用户可以给 Claude Code 安装插件和 Skill（可通过 UI 操作，无需命令行）。

建议：**一套代码跨平台，不要分别开发 Windows/macOS 原生应用**。优先选 **Electron + React + Node 后端壳层**，因为你的核心需求更像“轻量 VS Code + Lobe Chat + Claude Code Agent 管理器”。

## 推荐技术栈

| 层级       | 推荐                                 |
| -------- | ---------------------------------- |
| 桌面框架     | **Electron**                       |
| 前端       | React / Vue3 + TypeScript          |
| UI       | Ant Design / shadcn/ui / Tailwind  |
| 编辑器      | Monaco Editor                      |
| 文件树      | 自研 Tree + Node fs API              |
| Agent 接入 | Claude Code CLI / Claude Agent SDK |
| 本地服务层    | Electron Main Process + Node.js    |
| 配置存储     | SQLite / JSON 配置文件                 |
| 安装包      | electron-builder / Electron Forge  |
| 自动更新     | Electron autoUpdater               |

Electron 官方支持跨平台桌面应用、打包发布和 macOS/Windows 自动更新，适合做这类“桌面 IDE 壳”。([Electron][1])

## 各模块职责

### 1. 桌面壳层

负责普通软件体验：

```text
双击安装
桌面图标启动
自动更新
崩溃恢复
多窗口管理
系统托盘
```

用户不应该看到命令行，也不需要安装 Node.js、配置环境变量。

### 2. 对话模块

做成类似 Lobe Chat：

```text
左侧：会话列表
中间：聊天区
右侧：当前项目上下文 / 文件变更 / Agent 状态
底部：输入框、附件、执行按钮
```

对话不是直接调用普通大模型 API，而是把用户意图交给 **Claude Code Agent**。Claude Code 官方定位就是能读取代码库、编辑文件、运行命令并集成开发工具的 agentic coding tool。([Claude API Docs][2])

### 3. 文件管理模块

类似简化版 VS Code：

```text
打开文件夹
显示文件树
新建 / 删除 / 重命名文件
查看代码
编辑文件
保存文件
显示 Claude Code 修改前后 diff
```

建议第一版不要做完整 IDE，只做：

```text
文件树 + Monaco Editor + Diff Viewer
```

### 4. Claude Code Agent 适配层

这是核心，不建议前端直接调用 Claude Code。

应单独做一层：

```text
ClaudeCodeService
├─ startSession()
├─ sendMessage()
├─ stopTask()
├─ readStreamOutput()
├─ applyPermission()
├─ getDiff()
├─ rollbackChange()
└─ installSkill()
```

底层可以有两种接入方式：

| 方式                  | 适合阶段      |
| ------------------- | --------- |
| 调用 Claude Code CLI  | MVP 快速验证  |
| 使用 Claude Agent SDK | 后期更稳定、更可控 |

Claude Agent SDK 官方提供 Python 和 TypeScript 能力，可复用 Claude Code 的 agent loop、工具和上下文管理。([Claude API Docs][3])

### 5. 插件和 Skill 管理模块

UI 上提供：

```text
Skill 商店 / 本地安装
启用 / 禁用 Skill
查看 Skill 描述
安装进度
版本更新
卸载
```

底层本质是管理 Claude Code 的 Skill 目录、配置文件和依赖。Claude Code 官方支持创建、管理、共享 Skills。([Claude API Docs][4])

### 6. 设置模块

必须图形化：

```text
API URL
API Key
模型选择
代理设置
工作区权限
是否允许执行命令
是否允许修改文件
是否允许访问网络
```

Claude Code 本身有 settings 配置机制，你的桌面应用应把这些配置包装成表单。([Claude API Docs][5])

## 跨平台方案选择

### 推荐：Electron 一套代码跨平台

理由：

```text
1. 文件树、编辑器、聊天界面都适合 Web 技术实现
2. Monaco Editor、Markdown、Diff Viewer 生态成熟
3. Node.js 对文件系统、子进程、Claude Code CLI 调用更自然
4. Windows/macOS 打包、签名、自动更新方案成熟
5. 开发速度快，适合先做产品验证
```

### 不建议第一版用两套原生应用

原因：

```text
Windows 一套 C#/WPF
macOS 一套 SwiftUI

维护成本会翻倍：
- UI 逻辑重复
- Agent 适配重复
- 插件管理重复
- 更新机制重复
- Bug 修复重复
```

除非你后期已经验证商业价值，并且对系统原生体验要求极高，否则不值得。

### Tauri 是否可选？

可选，但我不建议第一版优先。

Tauri 体积更小、安全边界更好，也支持 sidecar 外部二进制打包。([Tauri][6])
但你的项目需要大量文件系统、编辑器、Agent 进程管理、插件安装、日志流式输出，Electron 的开发效率会明显更高。

## 关键技术难点与解决思路

### 难点 1：Claude Code 如何内置

不要让用户自己装 Claude Code。

建议：

```text
安装包内置 Agent Runtime
启动时检测运行环境
缺失组件自动初始化
所有配置写入应用私有目录
```

可以把 Claude Code 相关运行层作为“内置运行时”管理，用户只看到图形界面。

### 难点 2：Agent 执行命令的安全问题

非技术用户最危险的是误删文件、执行危险命令。

必须做权限系统：

```text
读取文件：默认允许
修改文件：需要确认
删除文件：强确认
运行命令：按命令类型分级
联网操作：可配置
安装插件：需要确认
```

建议 UI 上显示：

```text
Claude 准备修改 3 个文件
Claude 准备执行 npm install
Claude 准备删除 xxx 文件
```

用户点击“允许一次 / 始终允许 / 拒绝”。

### 难点 3：文件修改可回滚

必须内置 Git 或快照机制。

建议：

```text
打开项目时检测 Git
如果不是 Git 项目，自动创建本地快照
每次 Agent 修改前生成 checkpoint
修改后显示 diff
支持一键撤销
```

这对文科用户非常重要。

### 难点 4：一键安装

Windows：

```text
.exe / .msi 安装包
自动创建桌面图标
自动写入应用数据目录
自动更新
```

macOS：

```text
.dmg 安装包
拖入 Applications
应用签名 + notarization
自动更新
```

Electron 自带成熟打包和更新生态，官方也提供发布和自动更新流程。([Electron][7])

### 难点 5：插件和 Skill 安装

不能让用户输入命令。

UI 应该抽象成：

```text
搜索 Skill
查看说明
点击安装
安装成功
启用 Skill
```

底层做：

```text
下载 Skill
校验元数据
复制到 Claude Code Skill 目录
刷新配置
重启 Agent 会话
```

### 难点 6：对话体验不像命令行

Claude Code 原本偏终端体验，你要做一层“事件翻译”。

例如：

```text
Agent 正在读取文件 → UI 显示“正在分析项目结构”
Agent 修改文件 → UI 显示 diff 卡片
Agent 执行命令 → UI 显示可折叠日志
Agent 需要权限 → UI 弹出确认框
Agent 完成任务 → UI 总结修改内容
```

## 推荐落地版本路线

### MVP 版本

```text
Electron + React
打开文件夹
文件树
Monaco 编辑器
聊天界面
调用 Claude Code CLI
显示 Agent 输出
显示文件 diff
基础 API Key 设置
```

### V1 正式版

```text
权限系统
一键安装包
自动更新
会话记录
Skill 管理
Git checkpoint / 回滚
内置日志诊断
```

### V2 增强版

```text
Claude Agent SDK 深度接入
插件市场
项目模板
文科生友好的任务向导
可视化需求文档生成
一键生成网站 / 小程序 / 管理系统
```

## 最终建议

你的产品本质不是“Claude Code 图形壳”，而是：

```text
面向非程序员的 AI 软件开发工作台
```

第一版建议采用：

```text
Electron + React + Monaco Editor + Claude Code CLI/SDK + 本地权限系统
```
